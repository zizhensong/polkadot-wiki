其它程序语言实现

这里 Rust，C ++，Go 和 JavaScript 开发都是 Polkadot 不同语言的实现。

** Parity Polkadot **-Rust 客户端由 Parity 与他们在 Substrate 上的工作。

** Kagome ** - 由 Soramitsu 实现 C++ Polkadot runtime 环境 ，他们是日本的数字身份识别公司，之前开发了 Hyperledger Iroha 。他们获得了 Web3 基金会的资助，并计划在 2019年11月发布。他们正在用 C ++ 开发 libp2p 网络层。

** Gossamer ** - 由 ChainSafe Systems ，23人组成在多伦多的开发团队也正在开发 Eth2.0 Serenity 客户程序。拨款公告。

** Polkadot-JS ** - JavaScript 客户端和由 Polkadot JS 开发的工具。

为什么要把 Polkadot 实现多个语言？

如 Soramitsu 拨款公告所述:

由于多种原因，包括去中心化，知识分散和更好的协议定义，实现 Polkadot 协议极之 重要。实现多个 Polkadot 版本提高了网络的弹性，并增加了网络的去中心化。当多个团队在网络中运行开发节点的客户端时，网络的治理将更加民主化。
